    Fenella's latest comments for #4784 are really now tied to #4886:  The latest
symptom of seeing XML parsing error has been attributed to the problem where Get

corruption causing problems displaying messages.

workaround of deleting .msf files in the user profile directory(ies) will

in at least two places in the release notes.

the passwords are now stored in an encoded form.

"cookies".

counterpart under unix -- they are "cookperm" and "signon".  Besides adding
these two files to the unix table, you'll also need to update the wording below
the table which currently says "cookperm.txt is a file currently created for
windows, used to ...".  It should read "cookperm.txt is a file used to ...".
