    I see what the problem is. Body bgcolor is assigned a 'blue' value; table
bgcolor is assigned a 'white' value. Nav displays the table with a blue border.
Apprunner displays the table with a white border. The reason you see a
difference between Apprunner and Viewer is that Viewer is in 'Nav Quirks' mode.
In Viewer, go to the Style menu item and select Compatibility Mode
Pref/Standard. Now reload the page. The table should display with a white
border. If you want the table to have a blue border, then set the table bgcolor
value to blue or using Style Sheets and set border-color to 'blue'.

The correct rendering of the table in Gecko (in Standard mode) is for the border
to display in white. To be compatible with Nav, the table will display with a
blue border in 'Nav Quirks' mode.

The reference you made to bug #1163 is unrelated as that bug has an issue with
the attribute 'bordercolor' which is separate from the attribute 'bgcolor'.

I am marking this bug invalid.

    The border color of the table in both Standard and NavQuirks should be the same
since the <table> specifies a border and there is no css specifying a border
color. In this case, the color that is used is the background color of the body
since the table's background cannot be use (it is transparant) and the body is
the closest ancestor which has a non transparant background. The shade of blue
that is used seems to differ from Nav4.6, so if that is an issue please file a
new bug and assign it to gfx.

In NavQuirks mode, the background of the table applies to the cells but not the
space between the cells which is why there is no white space between the cells.
In Standard mode, the white background color of the table does show through the
cells. For a short time Apprunner incorrectly used Standard mode to display the
document but now it uses NavQuirks mode as the default.
