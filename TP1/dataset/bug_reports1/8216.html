    Using jun14 builds on Linux rh5.2, NT 4.0 and Mac OS 8.51

When you cancel a news message (and confirm the cancel), the message is not
removed from the thread pane. Even if you select another message or group and
return, the message appears listed in the thread pane. Even exiting and
returning doesn't do it.

Another issue (which may be part of bug #7778) is that the contents of the
message pane are not cleared until you make another selection then return to the
cancelled posting, at which time you'll see the "bad article" error message in
the message pane.

1.  Launch messenger using prefs specifying news server & group.
    I had one server specified, news.mozilla.org and used netscape.test
2.  Open netscape.test, select a message you posted.
    Edit|Delete Message.
Result:  message still appears in thread pane and the message contents still
display in message pane.  Select another message and find the cancelled message
is still displayed in the thread pane. It should not be.  Message header still
displays even after exit.
