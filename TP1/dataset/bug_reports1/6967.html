    Steps to Reproduce:
1) load www.nvidia.com
2) load some working address like www.yahoo.com
3) press Back button

Actual Results:
nothing happens (on browser view - some text on console)

Expected Results:
Site "www.nvidia.com" should have been showed on screen (it's back button:)

Build Date & Platform:
M6 (ID: 1999052308) - I think that the same bug has been around from the times
of M3 or so, but I thought at that time "back" wasn't implemented fully.

Additional Information:

This seems to be related to browser cache, because deleting cache directory and
then pressing back, the view refreshes. Cache seems to be corrupted(?) when
loading of document ends at "Error loading URL http://www.domain.com/". The
very same error message stops the parsing/rendering of page during loading (and
may be source of other rendering errors). I included also the last two lines of
my apache log in case it means something to you. File bg.jpg was loaded
successfully, but browser couldn't load bartop.gif (or at least it did not show
it).
"
- - [24/May/1999:18:07:21 +0300] "GET /mira/bg.jpg HTTP/1.1" 200 2897
- - [24/May/1999:18:07:21 +0300] "GET /mira/bartop.gif HTTP/1.1" 304 -
"
