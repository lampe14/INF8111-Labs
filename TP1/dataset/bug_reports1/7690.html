    I tried to build Mozilla 5 on my Solaris box and encountered a bug which is very
problematic, because this is a bug which may happen again which each Makefile
change... :(

The build-sequence was:
1. mkdir objdir
2. cd objdir
3. ../configure
4. make

After (a long) lunch I visited the results, and started "make" again - just for
fun...
...and the beast started to recompile MANY modules.

IMHO this is wrong !! A single "make"-run should cleanly do all jobs.

Later I figured out that "make -j 16" is broken (what else) :-(

Tested with:
- GNU Make version 3.76.1, by Richard Stallman and Roland McGrath.
- SunOS hackers_go_home 5.7 Generic_106541-03 sun4u sparc SUNW,Ultra-5_10
- egcs-2.91.66
- mozilla-1999-06-05-10-M7-source.tar.gz

This seems not to be a make-problem.
This is not a problem by my NFS-home-dir (xntp is working).
This bug is ********, because one change in the source tree causes that the
whole beast will be rebuild...
