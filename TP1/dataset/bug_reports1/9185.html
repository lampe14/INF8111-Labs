            Comment 11
          
    
          •
          
    21 years ago
          
    
            		
      
    

      
    
        
        
      
    
    
    

    
  
    The problem happens because we init appcores after we process command line args.
It is in appcore code that we first give the toolkit a chance to parse its args.
Gtk sort of assumes (as does Xt) that they are going to get a chance to look at
command line args early, and strip out the toolkit specific ones, and then
modify the argc, argv so that only application level args remain. A quick fix
would be to do what is being done in main1 for MAC, which is give gtk a chance
to parse out the toolkit args before calling into the appcode. Problem with that
is XP_UNIX is not always Gtk+, it may be Motif, etc..
There is a command line service. nsICmdLineService. Maybe it should be the one
responsible to determine if a toolkit function should be called first. And
perhaps that XP_MAC hack in main1() can be pushed down into implementation code
in the command line service too. I think perhaps a solution would be to add an
interface to the command line service that allows toolkits to look at the args
first, before anyone else.
