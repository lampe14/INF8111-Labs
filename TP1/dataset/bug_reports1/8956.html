    We need a centralize approach to focus management. Whenever you click on a
toolbar button (in Mail Composer, other UI elements?) the focus should return
to the editor window after the action is completed. This may not be simple,
since a dialog may popupup as part of the toolbar action, but we need some way
to return focus to the editor.
At a minimum, we need to be able to use the JS "focus()" method so we can use
that for every toolbar "OnFoo()" call, but currently the "contentWindow" we
get at editor startup doesn't return the correct window (the correct window is
a child of the only window we can get now: window.frames[0];
This issue is addressed by bug 7818.
This should also be a problem in the Browser.
