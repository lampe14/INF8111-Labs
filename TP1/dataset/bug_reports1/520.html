   Issue summary:                 missing registry entries

Issue details:
 In all earlier versions of Communicator, there was an entry
in

HKEY_LOCAL_MACHINE\SOFTWARE\NETSCAPE\NETSCAPE

NAVIGATOR\USERS pointing to the current user and subkeys for
each user profile

which contained an entry called "DirRoot" which in turn
pointed to the path where

prefs.js could be found permitting NSClean to locate
prefs.js to locate the files which

NSClean processes.



 We've since found "Biff" entries in HKCU but none point to
the location of the \users\

folder under Program Files and as you know, people can place
their Netscape

anywhere on the drive and will often relocate the \users\
folders. This in turn has

caused Privacy Software Corporation and its customers untold
expense and hassle.



 As a result, we are presently informing our customers to
AVOID the Netscape 4.5b1

product and are gravely concerned about how Netscape is
going to resolve this

missing data our product depends on.



 Are there any plans to correct this and provide a registry
pointer to where that folder

is located or do we need to talk to Jim B about it?

   Severity:                      Other
   Can reproduce?:                Yes, Always
   Try this URL:                  does not apply
   Additional computer info:       Any customer using
NSClean32 4.10 or NSClean16 3.35 products is affected.
