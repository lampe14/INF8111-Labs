 At any rate, I was looking at the page with viewer/win95. When I use
apprunner, I see what is posted in the first screen shot above for
usatoday.com.

 The bogus layout in both viewer and apprunner derives from this pattern.

  <NOBR>
  <TABLE WIDTH="132" BORDER="0" CELLPADDING="0" CELLSPACING="0" ALIGN="LEFT">
    ... left hand column, floated left ...
  </TABLE><TABLE WIDTH="468" BORDER="0" CELLPADDING="0" CELLSPACING="0">
    ... main body of story ...
  </TABLE>
  </NOBR>

 Remove the <NOBR>, and layout is fine (modulo one unrelated problem
concerning the line-height of empty <Hn></Hn> elements (the headline) --
I'll file that separately if it's not a duplicate).

 Attaching (1) the original page, (2) the original page minus the
<NOBR></NOBR> (layout is OK), and (3) a simple test case which shows a
textfield being placed *outside* the table in which it is defined.
