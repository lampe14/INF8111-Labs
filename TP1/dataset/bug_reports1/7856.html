    Let's try that again... The problem comes down to the way Mozilla handles floating
images inside table cells. If the cell is not wide enough to handle both the floating
image and any other content, other browsers will expand the width of the table cell
to allow both elements to appear next to each other. Mozilla, instead, pushes the
non-floating content down to below the floating image.

The rest of the layout problems in the first test case are simply consequences
of that one bug.

Is there a clear specification on what should happen in this situation? This bug
may need to be resolved for 4.X compatibility, but does HTML 4, or CSS define
what the correct behaviour should be here?

Am also not seeing any frame-related problems. Is this fixed now?
