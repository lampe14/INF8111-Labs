    This is a request to include support for CDE features, e.g. drag&drop and file
type assignment and helper applications defined in CDE should be available from
within Mozilla.
One example would be that the mime-type text/xml is defined within the CDE
config. Then Mozilla can get help infos and possible helper-applications, which
should appear in a menu (e.g. helper applications/tools for this attachment for
current viewed page).

    Here comes my TO-DO list for the Solaris version for Mozilla >V4:
- Get Motif Toolkit working (to make CDE's DnD and other things happy)
- Get printing via libXp (e.g. Xprt) working - this is _independant_ from Motif
toolkit (current setup is that libXp code is only build in conjunction with the
Motif toolkit)
- Fixing clipboard issues (like current selected text in Mozilla can't be put
into dtpad via middle-mouse button)

crashing CDE's dtfile -server, like the current Mozilla 4.7<->CDE1.3_dtfile
does - ahhgrrr...)

implemented to support CDE's style of bookmarks implemented.

are using /var/mail&&Mozilla 4.x (in traditional pop style (fetch messages from
/var/mail) and "new" imap stype (make references to /var/mail - to allow
concurrent use of dtmail, mozilla mail and /usr/ucb/from))...
- WE WANT WANT WANT tear-off menus :-)
- CDE-style skin...
- ... (and so on...)
