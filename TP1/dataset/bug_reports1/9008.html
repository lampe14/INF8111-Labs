    The page uses positioned divs and (implicitly) JS layer syntax to move the div.
The JS layer syntax is not supported in Raptor. The W3C DOM should be used
instead. The page should work if the function moveImage is changed as follows:

function moveImage() {
 if(document.getElementById) {
   divStyle = document.getElementById("mi").style;
 }
 else {
   if(!document.all) document.all = document;
   if(!document.all.mi.style) document.all.mi.style = document.all.mi;
   divStyle = document.all.mi.style;
 }
...
}

The page does not crash in the current build. Marking WONTFIX.

    This page should not work fully correctly on Nav5 because it uses document.all
(IE5 only) and document.layers[]. I think Jan reopened it because it fails to
show anything at all. In spite of the document.all/document.layers[] issue, one
*would* expect to see the two child pages of the top-level frameset displayed.

Problem might be related to the fact that this page has a custom onload method
that calls JavaScript wich will probably cause errors on Nav5.  So maybe this
bug should now be: "If onload method of page causes JavaScript errors, top-level
frameset children should still be loaded and displayed."

In that case, isn't this really a frameset bug now for pollmann?
