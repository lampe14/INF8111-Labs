    The problem here is that 4.x had all positioned elements (position of relative
or absolute, not fixed or static) with ID attributes as properties of the
document object.  That is, if you have:

<p id="myid" style="position: relative">
</p>

You can then refer to:

document['myid']

or

document.myid

to get the object for that paragraph.

Simpler test case to be attached.  In Nav 4.6, it shows alert "It worked!".  In
apprunner Linux 1999-08-20-13-M10, it gives JS error.

You should check MSIE5's behavior here too... (especially for position: fixed).

    This doesn't look like it but its in fact a layers bug.  Positioned elements are 
actually layers in 4.x and as such were added as document properties.  Since we 
don't have the same layers support in 5.0 this no longer works, nor will it.  IE 
has different support and this testcase only works there via browser sniffing.  
You'll notice the broken down test case does not.

The new standard DOM api's have call like GetElementByID which can do the same 
thing.  I'd suggest rewriting with those.
