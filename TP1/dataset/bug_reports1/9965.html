    Apparently, the grippy isn't showing up on Linux. Messenger in the closed state.

Use this pref if you want it open.

user_pref("sidebar.messenger.open", true);

This would get set if you clicked on it.  I'll make it so that the default is to
have it open.

    Build: 1999071911M9: Mac 8.5.1

The Sidebar on the Mac is always closed when opening Messenger.

Steps to Reproduce:
- After installing this build and opening Messenger, the Sidebar is closed by
default.
- Clicking on the grippy opens the Sidebar.
- I am able to open and close the Sidebar within a session.
- Open the Sidebar and close the application.
- Restart Messenger.

Actual Results: The Sidebar is closed.

Expected Results. The Sidebar should remember its last state. In this case it
should have been open.

Additional Builds and Platforms Tested On: On Linux and Windows both had the
Sidebar open by default. They also remembered the last state they were in.
