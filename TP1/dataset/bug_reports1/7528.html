    Hi,

Since you folks seem to be the main ones working on GTK+-related stuff, I figure
I'll assign this to you...

Anyway, I have been having a rather annoying problem here in apprunner.  My
problem is that I cannot view more than about 2 or 3 URLs before apprunner
crashes on me.  Unfortunately, I cannot give a definitive test case, since the
variables seem to change all the time, but anyway--here is the pertinent
information which might be of help:

My configuration is: Red Hat 6.0, glibc 2.1, GTK+ (tried with both 1.2.1 and the
latest 1.2.3), EGCS 1.1.2, and testing with Mozilla CVS builds just about
everyday for the past few weeks.

The main site that I try viewing where the crashes seem to occur is on
Slashdot.  Usually, after about 2 or 3 different page views, apprunner crashes.
I just started using GDB on Mozilla, and found in this instance:

Program received signal SIGSEGV, Segmentation fault.
theme_image_unref (data=0x0) at pixmap_theme_main.c:753
753       data->refcount--;

This source file, pixmap_theme_main.c, is from the gtk-engines package (the
package that provides GTK+ themes).  I will attach it to this bug in case you
don't have it and you want to examine it more carefully.

Unfortunately, I don't know if this is actually a GTK+ bug or a Mozilla bug,
because I do not know GTK+ well enough to say.  Do either of you??  It doesn't
seem to happen with any other GTK+ application I use...  And it happens even if
I don't use pixmap themes; it happens even on plain .gtkrc-type themes.  Any
ideas??

This might be related to my previously filed bug 7036 and bug 7037...  I don't
know, but since those bugs seem to be related to gtk themes as well, you might
want to consider this possibility.
