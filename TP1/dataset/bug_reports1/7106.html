    Here is the crux of the problem. We used to build a monolithic binary. To test
for success was very simple, looking for the executable. gmake error codes are
not always bubbled to the top-level gmake, so checking for error there is not
consistent. Since we live in a componentized world, it's very hard to check for
each of them. We might be able to hook up some kind of manifest file checking
thing based on the xpinstall stuff, but i'm not sure that is going to help us on
unix. I'm going to unset the target milestone and later this, because i can't
think of anything that doesn't involve a lot of manual work cataloguing what
components need to exist in order for the app to be considered `fully built.'
