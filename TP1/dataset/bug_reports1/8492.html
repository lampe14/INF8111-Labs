    There are major problems with the password entry screen which prevent
wallet/singlesignon being used at all on Mac and prevent them being used on Unix
unless you choose a blank password.

To reproduce on Mac:

1. Goto slip/projects/marvin/wallet/login.html
2. Enter any username/password and press Login
3. Choose OK to enable single signon if you this is your first sign-on
4. Choose OK to save this signon

Results: Password screen comes up but is totally blank.
Expected Results: Should get password entry screen with little comment about
hitting cancel if you want no password.

On unix, it actually works correctly the first time, but the second time you
come back and you are (supposed to be) asked for your password (which has
already been established) the password entry screen is yet again completely
blank.

This is using 6/18 builds. Windows works fine.

In addition, the wallet contents, display signons, safe form fill, and display
cookies dialogues layout blank on both Mac and Unix. I assume this is related,
but if not, let me know and a separate bug will be filed. You get errors at the
console on Unix that say "xp-connect-walleteditor.html failed". Same for
signonviewer.html, walletpreview.html and cookieviewer.html.
